INHERITANCE

Inheritance is a fundamental concept in object-oriented programming (OOP) that allows you to create new classes (derived or child classes) based on existing classes (base or parent classes). It enables code reuse, promotes modularity, and establishes a hierarchical relationship between classes.

In Java, inheritance is implemented using the extends keyword. The derived class inherits the properties (attributes) and behaviors (methods) of the base class, allowing it to extend or specialize the functionality of the base class.

Key concepts and features of inheritance in Java:

1.Superclass and Subclass:

The existing class, also known as the base class or superclass, serves as the foundation for creating new classes. The new class derived from the base class is called the derived class, subclass, or child class.

2.Code Reusability:

Inheritance allows you to reuse code from the base class, reducing code duplication and promoting modular design. The subclass automatically inherits all the accessible attributes and methods of the superclass, which can be directly accessed or overridden in the subclass.

3.Access Modifiers:

Inherited members (attributes and methods) retain their original access modifiers in the subclass. Public members can be accessed from anywhere, protected members can be accessed within the same package or subclass, and private members are not accessible in the subclass.

4.Method Overriding:

Inheritance allows a subclass to provide its own implementation of a method that is already defined in the superclass. This is achieved through method overriding, where the method in the subclass has the same name, return type, and parameters as the method in the superclass. The @Override annotation is used to indicate that a method is intended to override a superclass method.
